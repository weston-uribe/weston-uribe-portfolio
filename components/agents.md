# components/ — UI Components

## Purpose

All React components live here, organized into two subdirectories.

## Directory Structure

### `ui/` — shadcn/ui Primitives

These are auto-generated by the `npx shadcn add <component>` CLI. They wrap Radix UI primitives with Tailwind styling and follow the shadcn conventions.

**Rules:**
- Do not hand-edit these files. If you need to customize behavior, wrap the component in `custom/` instead.
- To add a new shadcn component, run `npx shadcn@latest add <name>` — it will be placed here automatically.

**Currently installed:** `button`, `card`.

### `custom/` — Project-Specific Components

Reusable components built specifically for Wedge-1. These compose shadcn primitives and project logic into higher-level UI building blocks.

**Rules:**
- Every component in `custom/` should be reusable across at least two contexts. If it is truly page-specific, co-locate it with the page instead.
- Use `cn()` from `@/lib/utils` for conditional class merging.
- Import design tokens from `@/lib/constants` rather than hardcoding spacing or layout values.
- Keep components focused: one responsibility per file.

## When to Create a New Component

- If you are duplicating JSX across two or more pages/components, extract it here.
- If a piece of UI has its own state or logic, it likely deserves its own component.
- Prefer composition: build complex UI by combining small components rather than making one large one.
